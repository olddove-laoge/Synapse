## Context

本变更面向“移动端 GraphRAG 第二大脑”MVP，目标用户先聚焦程序员群体（学习分布式系统、LLM 工程、复杂框架）。现状问题是输入碎片化、知识沉淀分散、跨时段回忆弱；传统大图谱交互在手机小屏不可用。技术上，手机端已具备 OCR、ASR、轻量 NER 的端侧推理条件，但复杂图推理与全局摘要仍需云侧支持。

关键约束：
- 小屏交互必须降低视觉复杂度，避免全局图谱渲染与误触。
- 涉及隐私文档时，原文默认不出端，端侧仅上传结构化摘要或脱敏实体。
- 用户输入来源多样（分享、识屏、语音），需要统一归一化入图协议。
- MVP 必须优先验证“可持续递归学习闭环”，而非大规模图谱性能上限。

## Goals / Non-Goals

**Goals:**
- 建立从碎片输入到图谱沉淀再到递归提问扩张的完整学习闭环。
- 提供移动端可用的 Focus View（当前节点 + 一阶邻居）与卡片式下钻体验。
- 构建端云协同流水线：端侧快速抽取，云侧深度推理，最终回流可解释结果。
- 提供系统级激活机制（Widget/通知）以提升知识复用与复习频率。
- 形成可执行 MVP 架构，支持后续多垂直场景扩展。

**Non-Goals:**
- 不在 MVP 中支持海量节点全景图编辑器与复杂手动布局。
- 不在 MVP 中提供跨团队协作、权限分级、企业级审计。
- 不在 MVP 中做通用搜索引擎替代或开放平台生态。
- 不在第一阶段追求离线全功能图推理。

## Decisions

1. 交互决策：采用 Micro-Graph Focus View 而非全局图谱
- 选择：每次只显示焦点节点及一阶关联，使用“点击节点 -> 平滑重心迁移 -> 新焦点扩展”的交互。
- 原因：移动端受限于屏幕和触控精度，全局图会显著降低可读性与可操作性。
- 备选：全局缩略图 + 手势缩放；否决原因是学习成本高、误触率高、信息噪声大。

2. 输入决策：统一三类入口到标准化 Ingestion Pipeline
- 选择：系统分享、识屏、语音统一进入同一解析管线，产出标准化节点草稿（实体、关系、来源、置信度、时间戳）。
- 原因：统一协议可降低后续去重、合并、追踪复杂度。
- 备选：各入口独立建模；否决原因是数据口径不一致，后续融合成本高。

3. 推理决策：端侧轻抽取 + 云侧重推理的混合 GraphRAG
- 选择：端侧执行 OCR/ASR/轻量 NER 与本地预摘要；云侧负责关系补全、社区发现、跨文档推理与全局摘要。
- 原因：兼顾隐私、时延、成本与效果；复杂推理在云侧更可扩展。
- 备选：全端侧；否决原因是模型与算力限制导致推理质量和迭代速度不足。
- 备选：全云侧；否决原因是隐私风险和上传延迟影响体验。

4. 成长决策：Chat to Graph 增量写入
- 选择：每轮问答后通过结构化提炼服务生成候选节点/关系，用户可一键确认入图。
- 原因：保证图谱增长与用户学习行为同步，形成可见的“认知扩张”反馈。
- 备选：仅保留聊天记录不入图；否决原因是难以复用与回顾，无法形成网络化知识资产。

5. 激活决策：系统级轻提醒驱动复习回路
- 选择：每日未探索枝桠提醒 + 新旧节点潜在关联推送。
- 原因：把“记得回来学”从用户自律转为系统主动服务。
- 备选：仅在 App 内提醒；否决原因是触达弱，难形成长期习惯。

## Risks / Trade-offs

- [端云职责边界漂移] 模块职责变得含混，影响稳定性 -> 通过明确接口契约（输入输出 schema、SLA、失败重试策略）约束。
- [端侧抽取精度不足] 影响入图质量 -> 引入置信度阈值与人工确认模式；低置信内容进入待审核池。
- [通知过多造成打扰] 可能导致关闭通知 -> 增加频控与相关性阈值，支持用户自定义提醒强度。
- [图谱持续增长导致性能压力] 影响焦点视图响应 -> 采用一阶邻居按需加载、分层缓存与后台索引维护。
- [隐私与合规风险] 本地敏感信息上云争议 -> 默认最小化上传，提供“仅端侧处理”策略开关与审计日志。

## Migration Plan

1. 第一阶段（MVP）
- 完成单文档/PDF（<=10 页）解析、基础节点构建、Focus View 展示、对话问答与增量入图。
- 支持系统分享入口与语音随记入口。

2. 第二阶段（Beta）
- 增加跨文档关系推理、社区发现、节点聚类摘要。
- 加入 Widget 未探索枝桠提醒与关联通知。

3. 第三阶段（扩展）
- 拓展到更多垂直场景（医学生、金融研究）并评估场景模板化能力。

回滚策略：
- 若云侧推理异常，降级为端侧基础抽取 + 本地问答；保留核心录入和浏览能力。
- 若通知策略效果不佳，关闭主动推送，仅保留 App 内回顾入口。

## Open Questions

- 是否在 MVP 即支持多模态输入（图片+语音组合）的联合实体抽取，还是先分步处理？
- 程序员场景的 Aha 指标优先定义为“问题解决速度提升”还是“知识复用率提升”？
- 端侧模型在不同机型上的最小可用性能基线如何定义与监控？
- 在系统分享链路中，如何平衡“一键入图效率”与“入图质量可控性”？
